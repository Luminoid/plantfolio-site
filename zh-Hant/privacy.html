定義

就本政策而言：

  • 「個人資料」指與已識別或可識別的自然人有關的任何資訊。
  • 「處理」指對個人資料實施的任何操作或一組操作（如蒐集、儲存、變更、揭露）。
  • 「資料控制者」指決定處理目的和方式的自然人或法人。
  • 「使用者資料」指使用者在本應用中輸入、產生或儲存的所有資料，包括但不限於植物資料、照片、養護記錄、植物集及設定。

我們不營運任何伺服器，也不在我們控制的系統上儲存、保留或處理任何使用者資料。所有使用者資料僅保留在使用者裝置上，或在使用者已啟用 iCloud 同步時保留於 Apple Inc.（「蘋果」）營運的 iCloud 服務基礎架構上（與使用者的 Apple ID 關聯）。我們無法存取、也不接收或傳輸使用者資料。

1. 所處理資料的類別

1.1 使用者提供的資料

在使用本應用過程中，使用者可能輸入或提供以下類別的資料，這些資料僅在使用者裝置上本機處理，或在啟用 iCloud 同步時由蘋果代表使用者處理：

  • 植物與植物集資料：名稱、學名、植物類型、描述、研究筆記、光照與土壤偏好、盆器材質、季節性澆水週期、開始日期。
  • 照片：新增到本應用的植物或植物集圖片，可帶分類標籤（如生長、開花、問題）。
  • 發現資料：新增到本應用的植物發現記錄，包括照片、可選名稱或備註以及 GPS 座標（經緯度）。GPS 座標可能來自照片的 EXIF 中繼資料或建立發現記錄時裝置的位置。
  • 養護記錄：澆水日誌（日期、時間、強度與備註）、生長測量（高度、寬度、備註）、施肥記錄（類型、NPK、用量、方式、備註）、修剪記錄（類型、手法、備註）、植物狀態檢查報告（AI 產生的健康評估與對話歷史）、澆水改期或延後。
  • 植物集結構：植物歸入植物集的方式及每株植物至多屬於一個植物集的從屬關係。
  • 應用設定：日期格式、語言偏好（如英文、中文、西班牙文）、通知偏好（開/關、提醒時間）、預設排序與篩選選項、自訂土壤或盆器類型、是否隱藏已封存植物、所選 Gemini AI 模型偏好。

1.2 應用自動取得的資料

以下資料可能僅由本應用在使用者裝置上取得或衍生；該等資料均不會向我們傳輸：

  • 照片中繼資料：可能讀取 EXIF 資料（如拍攝日期）或照片資源的建立日期以自動填入日期欄位。該處理僅在使用者裝置上進行。
  • 位置資料：當使用者新增發現記錄時，本應用可能從照片 EXIF 資料中讀取 GPS 座標，若 EXIF 中無位置資訊，則在使用者授權下從裝置目前位置取得。該座標僅儲存在使用者裝置上（以及在啟用 iCloud 同步時儲存在蘋果 iCloud 基礎架構上）。我們不接收也不傳輸位置資料。
  • 日期與季節：可能使用裝置日期與行事曆判斷目前季節（用於澆水計畫）及目前日期（用於提醒與行事曆檢視）。不會向我們傳輸位置或時區資料。
  • 裝置資訊：蘋果的 App Store 及系統服務可能根據蘋果政策蒐集資訊（如 iOS 版本、裝置型號）。我們不使用第三方分析或追蹤。

1.3 不蒐集的資料

我們不蒐集，且本應用亦非為蒐集以下內容而設計：

  • 個人身分資訊（姓名、電子郵件、實體地址），除非使用者在聯絡我們時自願提供（如支援諮詢）。
  • 精確或大致位置資料，但使用者透過發現功能自願記錄的 GPS 座標除外，該資料僅保留在使用者裝置上（以及在啟用 iCloud 同步時保留在蘋果 iCloud 基礎架構上）。
  • 付款或財務資訊（本應用不處理付款）。
  • 本應用外的瀏覽歷史或活動。

2. 處理目的與法律依據

2.1 目的

使用者資料僅用於以下目的而處理：

  • 提供並維護本應用核心功能（植物與植物集管理、行事曆、澆水計畫、照片、養護日誌、植物狀態檢查、及發現與地圖檢視）。
  • 啟用可選功能，如澆水提醒、行事曆檢視、植物整理、以及透過 Google Gemini API 提供的 AI 植物健康評估（當使用者發起狀態檢查時）。

當使用者為支援目的聯絡我們（如透過電子郵件）時，我們僅處理該通訊中所載或所附資訊以回覆諮詢。

2.2 法律依據（歐盟/英國 GDPR）

在與本應用相關的個人資料處理範圍內，該處理僅在使用者裝置上或在蘋果營運的基礎架構上（當啟用 iCloud 同步時）進行。我們不在我們控制的系統上處理個人資料。就本應用在使用者裝置上可能被視為處理個人資料而言，依據《一般資料保護規則》（歐盟）2016/679 及英國 GDPR 的法律依據為：

  • 合約履行（第 6(1)(b) 條）：處理為履行使用者與我們之間的合約（提供本應用及其核心功能）所必需。

3. 儲存與安全

3.1 資料存放位置

我們不儲存使用者資料。所有使用者資料僅存放在以下一處或兩處：

  • 使用者裝置：使用者資料——包括植物資料、照片、發現位置及養護記錄——透過 SwiftData 框架本機儲存在使用者裝置上。該資料保留在裝置上，不會向我們傳輸。
  • 蘋果 iCloud（可選）：當使用者已啟用 iCloud 同步時，本應用使用 SwiftData 的 CloudKit 整合，使使用者資料的副本亦儲存在蘋果營運的 iCloud 基礎架構上，與使用者 Apple ID 關聯。我們無法存取該資料。

3.2 iCloud 同步

當啟用 iCloud 同步時，使用者資料可能儲存在使用者裝置及蘋果 iCloud 基礎架構上。該資料：

  • 由蘋果 iCloud 服務在傳輸與靜態時加密。
  • 受蘋果 iCloud 條款與條件及蘋果隱私政策約束。
  • 僅使用者在使用其 Apple ID 登入的裝置上可存取。
  • 我們或我們控制的任何第三方無法存取；我們不營運伺服器也不儲存使用者資料。

iCloud 同步為可選，可隨時透過本應用或裝置系統設定關閉。

3.3 技術與組織措施

使用者資料的安全由其存放位置決定：

  • 在使用者裝置上：資料透過 iOS 安全儲存機制（SwiftData）儲存。
  • 當啟用 iCloud 同步時：資料經加密後儲存在蘋果 iCloud 基礎架構上。
  • 不會向我們或我們營運的任何系統傳輸使用者資料。

4.2 Google Gemini API

本應用提供可選的 AI 功能——植物辨識、養護自動填入、植物資訊更新和植物狀態檢查——使用 Google 的 Gemini API。當使用者啟動上述功能時：

  • 傳送至 Google 的資料：根據所使用的功能，以下資料可能包含在傳送至 Google Gemini API 的請求中：
    • 植物照片（1–6 張，壓縮後的 JPEG）
    • 植物名稱與學名（作為上下文提示）
    • 植物集中各植物的名稱（在辨識或評估包含多株植物的植物集時）
    • 盆器材質、土壤層次與位置/氣候描述（僅文字，非 GPS 座標）
    • 辨識時儲存的物種偏好（光照、濕度、溫度範圍、排水性、土壤 pH、澆水方式）及生長條件（半球）
    • 目前季節澆水間隔（春、夏、秋、冬）
    • 近期養護歷史：澆水日誌（最近 20 條，含日期、操作與強度）、施肥記錄（最近 3 條，含類型、NPK 與方式）、修剪記錄（最近 3 條，含類型）、生長測量（最近 3 條，含高度與寬度，用於狀態檢查）
    • 使用者備註與此前 AI 備註（用於狀態檢查與植物資訊更新）
    • 追問對話的對話歷史（用於多輪狀態檢查；對話結束後不保留）
    • 照片日期、照片歷史中繼資料（已儲存照片的數量與日期範圍）及使用者的語言偏好
    不會傳送 GPS 座標、帳號資訊或其他植物/功能的資料。
  • Google 的處理：Google 處理請求並回傳結果（辨識結果、養護建議或健康評估）。Google 的處理受 Google Gemini API 服務條款和 Google 隱私政策約束。
  • 結果本機儲存：AI 產生的結果（辨識備註、狀態檢查報告、對話歷史）儲存在使用者裝置上，以及在啟用 iCloud 同步時儲存在蘋果 iCloud 基礎架構上。我們不儲存也無法存取這些結果。
  • 僅使用者發起：除非使用者明確觸發辨識、更新或狀態檢查操作，否則不會向 Google 傳送任何資料。使用者的 Gemini API 金鑰儲存在使用者裝置上的應用設定中。

4.3 匯出與匯入

本應用允許使用者將使用者資料（植物、植物集、照片、澆水與養護日誌、設定）匯出至使用者裝置上的 ZIP 檔案或「檔案」應用，並從先前匯出的 ZIP 匯入（全部取代或合併）。匯出與匯入僅在使用者裝置上或使用者選擇的位置進行。我們不接收、儲存或處理匯出或匯入的資料。

4.4 外部連結與裝置內參考資料

本應用可能包含指向第三方資源（如維基百科）的連結以取得植物養護資訊。存取該等連結會使使用者離開本應用；該第三方的隱私做法與條款適用。本應用還包含儲存在裝置本機的內建參考資料（如植物類型與養護資訊）；該資料不用於識別使用者。

4.5 共享擴充功能

本應用包含共享擴充功能，允許使用者從其他應用（如照片、相機）直接將圖片共享到本應用中。透過共享擴充功能接收的圖片由相關應用透過 iOS 標準共享面板機制傳輸至本應用，並儲存在使用者裝置上的應用共享容器中。該圖片僅在使用者裝置上處理，不會傳輸至我方。

4.6 Spotlight 搜尋索引

植物、植物集和發現名稱透過 CoreSpotlight 索引到裝置的 Spotlight 搜尋索引中，以便透過系統搜尋快速尋找。Spotlight 索引僅在使用者裝置本機進行；不會向我方傳輸任何搜尋資料、索引項目或查詢內容。所有索引內容均保留在使用者裝置上。

11. 保留與外洩

11.1 保留

我們不保留使用者資料，因我們不儲存該資料。使用者資料僅保留在使用者裝置上，以及在啟用 iCloud 同步時保留在蘋果 iCloud 基礎架構上。保留期限由使用者決定：資料在本應用中刪除或解除安裝本應用時從裝置移除；蘋果基礎架構上的資料將保留至使用者透過 Apple ID 設定刪除。

11.2 自動化決策

我們不從事 GDPR 第 22 條所指的、對使用者產生法律效力或同樣重大影響的自動化決策或剖析。澆水計畫計算及「下次到期」等功能基於使用者提供的資料與裝置日期，並在裝置本機執行；不構成該等自動化決策。

11.3 個人資料外洩

若發生可能對自然人權利與自由造成風險的個人資料外洩，我們將在適用法律要求時：

  • 在外洩可能對資料主體權利與自由造成高風險時，毫不延遲地通知受影響的資料主體。
  • 通報外洩性質及已採取或擬採取的措施。

我們不營運伺服器也不儲存使用者資料。所有使用者資料保留在使用者裝置或蘋果 iCloud 基礎架構上（當同步啟用時）。因此我們不持有可能因我方系統外洩而受影響的使用者資料；前述義務僅在我們處理或持有個人資料的範圍內適用。
